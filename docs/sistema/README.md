# sistema

Esta pasta reúne documentação estrutural de apoio do Cérebro Tendoshk.

Ela existe para organizar peças que explicam módulos, integração, governança, mapas, incidentes e funcionamento geral do sistema, sem concorrer com a soberania executiva dos `masters`.

## Papel desta pasta
Aqui ficam documentos de:
- integração entre cérebro, NPT e camadas correlatas
- módulos e especificações operacionais
- governança documental
- mapas estruturais
- registros técnicos auxiliares
- incidentes relevantes para entendimento do sistema

## Regra de autoridade
Os arquivos desta pasta servem como:
- apoio documental
- referência estrutural
- contexto técnico
- memória explicativa do sistema

Eles **não substituem**:
- `docs/masters/`
- `docs/checkpoints/`

## Regra prática
Esta pasta explica e organiza.
Ela não governa a execução final dos comandos específicos.
