# HISTÓRICOS DE MASTERS — Cérebro Externo Tendoshk

## Natureza

Esta pasta existe para preservar, de forma rastreável, versões anteriores de masters soberanos do sistema quando houver transição estrutural relevante.

Ela não é um depósito genérico de cópias antigas.  
Ela é uma trilha documental de evolução dos masters.

## Função

Esta pasta existe para permitir que o sistema preserve:

- a formulação anterior de uma peça soberana
- a linhagem evolutiva da regra
- o motivo da transição
- o ponto em que uma versão deixou de ser canônica
- a rastreabilidade entre versão histórica e versão vigente

## Regra de uso

Esta pasta deve ser usada quando ocorrer **transição soberana** de um master.

Isso inclui casos como:

- refinamento estrutural relevante
- mudança de dependência entre peças
- criação de nova amarração forte
- divisão de uma regra em mais de uma peça
- correção de ambiguidade arquitetural
- mudança na governança operacional de uma peça soberana

## Regra crítica

Nem toda alteração em master exige histórico.

Não usar esta pasta para:

- correções cosméticas
- ajustes de redação sem impacto estrutural
- correção ortográfica
- microclareza sem mudança de governança
- alterações triviais sem efeito na arquitetura do sistema

## Critério correto

Usar esta pasta quando a versão anterior ainda possuir valor de auditoria, linhagem, comparação, governança ou compreensão histórica da evolução do sistema.

## Relação com o master vigente

Os arquivos desta pasta **não são a versão canônica ativa**.

A versão canônica vigente continua sempre em:

`docs/masters/`

Esta pasta guarda apenas a formulação histórica preservada da peça.

## Convenção de nomes

Os arquivos históricos devem seguir, preferencialmente, este padrão:

`<nome_do_master>__AAAA_MM_DD_HH_MM__<motivo_curto>.md`

Exemplo:

`presenca_viva__2026_04_15_10_40__antes_da_amarracao_exclusiva_ao_tgm.md`

## Regra de integridade

Sempre que possível, a transição deve deixar claro:

- qual era a versão anterior
- qual passou a ser a versão canônica
- por que a transição ocorreu
- quais peças correlatas foram afetadas

## Regra final

Histórico não é ruído.  
Histórico é continuidade rastreável da evolução soberana do sistema.
